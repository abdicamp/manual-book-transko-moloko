Sistem Monitoring NextSense – Transko Moloko

Versi Dokumen 1.0 | PTK System – NextSense

1. Pendahuluan

Sistem Monitoring NextSense – Transko Moloko digunakan untuk memantau kondisi operasional kapal secara real-time dan historis melalui data sensor mesin dan Door Panel. Sistem ini mendukung kebutuhan operasional, evaluasi performa mesin, serta dokumentasi dan audit.

2. Halaman Login

Halaman Login merupakan gerbang utama untuk mengakses Sistem Monitoring NextSense. Setiap pengguna wajib melakukan autentikasi untuk memastikan bahwa hanya personel yang berwenang yang dapat mengakses data operasional kapal.

Pada halaman ini, pengguna diminta untuk memasukkan:
  • Username – identitas pengguna yang telah terdaftar di sistem
  • Password – kata sandi rahasia yang bersifat pribadi
Setelah data dimasukkan dengan benar, klik tombol Login. Jika autentikasi berhasil, sistem akan mengarahkan pengguna ke Dashboard Utama. Apabila gagal, sistem akan menampilkan pesan kesalahan.
Gambar 1. Halaman Login NextSense

3. Dashboard Utama

Dashboard Utama adalah halaman inti yang menampilkan ringkasan kondisi operasional kapal secara real-time. Informasi yang ditampilkan pada dashboard bertujuan untuk memberikan gambaran cepat kepada operator mengenai status mesin dan sensor.

Pada halaman ini, pengguna dapat melihat:
  • Status LIVE yang menunjukkan bahwa sistem sensor terhubung dan mengirimkan data
  • Ringkasan RPM Mesin untuk memantau performa mesin secara langsung
  • Status Door Panel untuk mengetahui kondisi pintu kapal (OPEN / CLOSED)
Dashboard dirancang agar mudah dipahami sehingga operator dapat segera mengambil tindakan apabila terjadi kondisi abnormal.
Gambar 2. Dashboard Live Sensor Data

4. Monitoring RPM Mesin

Fitur Monitoring RPM Mesin digunakan untuk memantau kecepatan putaran mesin kapal secara kontinu. Data RPM sangat penting untuk memastikan mesin beroperasi dalam batas normal dan untuk mendeteksi potensi gangguan sejak dini.

Sistem menampilkan grafik RPM untuk beberapa mesin, antara lain:
  • RPM AE 2 – mesin auxiliary engine 2
  • RPM AE 3 – mesin auxiliary engine 3
  • RPM ME Portside – mesin utama sisi kiri
  • RPM ME Starboard – mesin utama sisi kanan
Grafik ditampilkan berdasarkan waktu sehingga pengguna dapat menganalisis tren kenaikan atau penurunan RPM selama operasional kapal.
Gambar 3. Grafik RPM Mesin

5. Monitoring Door Panel

Monitoring Door Panel berfungsi untuk mencatat dan menampilkan status pintu kapal secara real-time dan historis. Informasi ini penting untuk aspek keselamatan dan keamanan operasional kapal.

Status pintu ditampilkan dalam dua kondisi utama:
  • OPEN – pintu dalam kondisi terbuka
  • CLOSED – pintu dalam kondisi tertutup
Grafik Door Panel memungkinkan operator melihat kapan pintu dibuka atau ditutup dalam periode waktu tertentu.
Gambar 4. Grafik Door Panel

6. Data Historis & Filter

Fitur Data Historis & Filter digunakan untuk menampilkan data sensor pada periode waktu tertentu. Fitur ini sangat berguna untuk analisis performa mesin, investigasi kejadian, dan keperluan pelaporan.

Pengguna dapat menentukan:
  • Tanggal dan waktu awal
  • Tanggal dan waktu akhir
Setelah filter diterapkan, sistem akan menampilkan grafik sesuai dengan rentang waktu yang dipilih.
Gambar 5. Filter Date & Sensor

7. Export Laporan

Fitur Export Laporan digunakan untuk menyimpan data monitoring dalam bentuk dokumen yang dapat digunakan sebagai arsip, laporan manajemen, maupun keperluan audit.

Sistem menyediakan dua format export:
  • PDF – digunakan sebagai laporan resmi yang siap dicetak
  • Excel – digunakan untuk analisis lanjutan dan pengolahan data
Sebelum melakukan export, disarankan untuk menerapkan filter tanggal agar data yang dihasilkan sesuai kebutuhan.
Gambar 6. Tombol Export PDF & Excel

8. Alur Operasional Harian Kapal

Alur operasional harian digunakan sebagai panduan standar bagi operator dalam memanfaatkan Sistem Monitoring NextSense selama kegiatan kapal berlangsung.
  1. Operator melakukan login ke sistem NextSense
  2. Pastikan status koneksi sensor dalam kondisi LIVE
  3. Melakukan pemantauan RPM mesin secara berkala
  4. Memastikan status Door Panel sesuai prosedur keselamatan
  5. Melakukan pengecekan data historis apabila terjadi anomali
  6. Admin melakukan export laporan harian atau periodik

9. SOP Kondisi Abnormal & Troubleshooting

Bagian ini menjelaskan langkah-langkah yang harus dilakukan apabila sistem atau data sensor menunjukkan kondisi tidak normal.

9.1 Sensor Tidak LIVE

  • Periksa koneksi jaringan kapal
  • Pastikan perangkat sensor dalam kondisi menyala
  • Refresh halaman dashboard
  • Apabila masalah berlanjut, hubungi Admin atau Tim Teknis

9.2 RPM Mesin Tidak Normal

  • Perhatikan lonjakan atau penurunan RPM secara tiba-tiba
  • Bandingkan dengan data historis pada periode sebelumnya
  • Laporkan kepada bagian teknis kapal jika nilai berada di luar batas operasional

9.3 Door Panel Tidak Sesuai Kondisi Aktual

  • Pastikan kondisi fisik pintu kapal
  • Periksa sensor Door Panel
  • Catat kejadian untuk keperluan laporan

9.4 Gagal Export Laporan

  • Pastikan rentang tanggal sudah dipilih
  • Periksa koneksi internet
  • Coba ulang proses export
